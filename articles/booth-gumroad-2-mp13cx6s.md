## 出品ボタンの前で止まるのは「作業力」の問題ではない

デジタルコンテンツは完成している。  
価格も決めた。サムネイルも用意した。  

それでも、出品ページが公開できない。

原因はほぼ毎回同じだ。

**「説明文、なんて書けばいいか分からない」**

これは能力の問題ではない。「言葉の設計」に慣れていないだけ。商品を作った本人が最も陥りやすいのが、「自分には当たり前すぎて、読者の視点で書けない」という状態だ。

BOOTH・Gumroadに初出品する人が詰まりやすいポイントは、主に3つある。

1. **商品説明文**：「何が入っているか」は書けるが、「使うと何が変わるか」が書けない
2. **LPのコピー**：キャッチコピーが思い浮かばず、ページ全体が止まる
3. **SNS告知文**：「売り込みっぽくなるのが嫌で投稿できない」

3つとも、言葉の問題。作業の問題ではない。

そして言葉の問題は、AIに下書きを出させることで、ゼロから書く負担を大きく減らせる可能性がある。

この記事では、デジタル商品「はじめての出品ことばキット」から2種のプロンプトを抜粋して公開する。どちらもChatGPT・Claude・Geminiにそのままコピペして使えるよう設計されている。

## 無料サンプル①：テンプレ系商品（Notion・スプレッドシートなど）の説明文プロンプト

テンプレ系商品の説明文は「機能の羅列」になりがちなのが難点。読んでいる側が「で、私にとって何が嬉しいの？」と感じてしまうパターンが多い。

このプロンプトは「購入者が使った後の状態」から説明文を組み立てる構成になっている。

```
あなたはBOOTHで販売するデジタル商品の説明文ライターです。
以下の商品情報をもとに、購入者が「これを使うと何が変わるか」を
中心にした説明文を書いてください。

【商品名】○○テンプレート
【対象読者】（例：フリーランスの案件管理が煩雑な人）
【商品の内容】（例：Notionで使える案件管理テンプレート、○ページ構成）
【購入者が抱えている悩み】（例：毎回ゼロからスプレッドシートを作り直している）
【商品を使った後の状態】（例：テンプレートを複製するだけで管理をすぐ始められる）
【価格】¥○○○

出力形式：
1. キャッチコピー（30字以内）
2. リード文（3〜4文、読者の悩みから入る）
3. 「こんな人向け」箇条書き（3〜5項目）
4. 「含まれるもの」箇条書き
5. 購入後すぐできること（1〜2文）

表現ルール：
- 「必ず」「誰でも」「一瞬で」「爆速」などの効果保証表現は使わない
- 事実ベースで書く。できることを書き、できないことは書かない
- 読みやすさ重視。箇条書きと短文を使う
```

「表現ルール」のセクションを入れているのがポイント。ここに制約を明示しておかないと、AIは誇大表現を平気で出力する傾向がある。プラットフォームの規約的にも、購入者との信頼関係的にも、事実ベースの説明文にしておくほうが長期的に安定する。

## 無料サンプル②：プロンプト集系商品の説明文プロンプト

AIプロンプトを商品として販売する場合、「プロンプトが入っています」だけでは読者に価値が伝わらない。「何の作業が省けるのか」まで書いて、初めて読者の判断材料になる。

```
あなたはBOOTH・Gumroadで販売するプロンプト集の説明文ライターです。
以下の情報をもとに、購入者が「このプロンプトで何の作業が省けるか」を
伝える説明文を書いてください。

【商品名】○○プロンプト集
【プロンプトの用途】（例：Instagram投稿キャプション生成、営業メール下書き作成）
【収録数】○本
【対象読者】（例：毎日SNS投稿に時間がかかるフリーランス）
【使い方の流れ】（例：プロンプトをコピー → 自社情報を入力 → AIに送信）
【価格】¥○○○

出力形式：
1. キャッチコピー（25字以内）
2. リード文（2〜3文）
3. 「収録プロンプト一覧」の見出しと3〜5項目のサンプル
4. 「こんな作業に使える」ユースケース箇条書き
5. 注意事項（例：AIのアカウントが別途必要など）
```

このプロンプトで重要なのは「注意事項」の出力指定を入れていること。「AIのアカウントが別途必要」「特定ツールのバージョンが必要」などを説明文に事前記載しておくことで、購入後の認識ズレを減らす効果が期待できる。説明文はコンテンツの紹介文であると同時に、購入後の体験設計の一部でもある。

## 使い方とカスタマイズの考え方

この2つのプロンプトに共通する構造は、**「入力 → 出力形式の指定 → 表現ルールの制約」** の3層になっている。

操作手順はシンプル。

1. プロンプトをコピーする
2. `【　】` で囲まれた部分を自分の商品情報に書き換える
3. ChatGPT・Claude・Geminiのいずれかに貼って送信する

AIが下書きを出力したら、事実確認と語感の微調整だけ行えばよい。

カスタマイズで最も効果的なのは「出力形式」を変えること。たとえば、Gumroadのページはリード文を短くしてメール登録を促す構成にしたい、BOOTHは含まれるものの一覧を詳細に書きたい、といった用途の違いは「出力形式」の指示を書き換えるだけで対応できる。プロンプト本体を大きく変える必要はない。

「表現ルール」も同様に調整可能。感情訴求を強めたい商品は制約を緩め、事実だけで勝負したい商品は制約を厳しくする。このチューニングの余地があるのが、テンプレート形式のプロンプトを使う利点のひとつだと感じている。
