## 評価コメントに30分以上かけていないか

期末・中間評価の時期になると、こんな状況になりやすい。

- 「良かったとは分かっている。でも言葉にならない」
- 「去年と同じ文章になってしまった」
- 「厳しいことを書きたいが、傷つけたくない」

評価コメントの悩みは「内容が分からない」より「言語化できない」ケースがほとんどだ。  
管理職は現場で何が起きているかを知っている。足りないのはその事実を適切な文章に変換する手間と時間だ。

ChatGPTを使えば「ゼロから書く」という消耗をある程度減らせる可能性がある。ただし、プロンプトの設計が甘いと出力がぼんやりした文章になる。職種・評価段階・評価項目という3軸で指示を絞ることで、実際に使える下書きが出てきやすくなる、というのが筆者の体感だ。

## 評価コメント作成の基本フロー

プロンプトを使う前に「事実メモ」を用意しておくと、出力の精度が上がりやすい。

```
【事実メモ テンプレート（コピーして使う）】

・対象者の職種・役割：
・評価期間（例：2024年10月〜2025年3月）：
・主な担当業務：
・印象に残った出来事（良い点）：
・改善してほしい点：
・目標に対する達成状況（〇〇%達成、など主観で構わない）：
・来期に期待すること：
```

この箇条書きをプロンプトに貼り付けて送るだけでよい。評価の判断は管理職が行う。ChatGPTはあくまで「下書きを出す役割」として使う。

## 無料サンプル①：全職種共通｜基本プロンプト「事実→評価→期待」の型

まずは職種を問わず使える汎用プロンプトから。

```
あなたは人事評価コメントの作成を支援するアシスタントです。
以下の情報をもとに、管理職が部下に渡す人事評価コメントの下書きを書いてください。

【対象者の職種・役割】
（ここに記入）

【評価期間】
（例：2025年4月〜9月）

【事実ベースのメモ（箇条書きで構いません）】
（ここに記入）

【評価段階】（良い／標準的／改善が必要、のいずれかを選ぶ）
（ここに記入）

【出力の条件】
- 「事実→評価→期待」の3段構成で書く
- 250字以内
- 本人が読んで納得できる表現にする
- 「いつも」「絶対に」「必ず」などの断定表現は避ける
- 人格・性格への言及は避け、行動・成果について書く
```

**Before / After の比較**

| | 内容 |
|---|---|
| **Before（よくあるパターン）** | 「今期も頑張っていました。来期も期待しています。」（情報ゼロ） |
| **After（プロンプト使用後）** | 「新規顧客へのアプローチ件数を前期比で増やし、うち複数案件を受注につなげた。顧客の課題を丁寧にヒアリングする姿勢が評価できる。来期は既存顧客のフォロー頻度を高め、継続受注の安定化に取り組んでほしい。」 |

After側は具体的な行動・成果・期待が揃っている。情報がなければこうはならないが、事実メモがあれば近い出力が出てきやすい。

## 無料サンプル②：経理職｜改善が必要な場合のコメント

厳しい評価ほど言語化が難しい。「改善を求めながら、相手の尊厳を損なわない」という条件をプロンプトに明示するのがポイントだ。

```
以下の条件で、経理・財務職に向けた人事評価コメントの下書きを書いてください。
改善を求める内容ですが、相手の尊厳を損なわない表現にしてください。

【職種】経理・財務
【評価期間】（記入）
【事実ベースのメモ】
（記入）

【重点評価項目】
- 締め処理の正確性・スピード
- 社内外への数字の説明能力
- ミスが発生した場合の再発防止への取り組み

【出力の条件】
- 改善点は「〜という点で、さらに精度を高めることを期待している」など
  前向きな言い回しに変換する
- 責めるトーンは避け、「次期への期待」で締める
- 200〜300字で出力する
- 人格・属性への言及は行わない
```

「人格への言及を避ける」「前向きな言い回しに変換する」という条件をプロンプトに書いておくと、ハラスメントリスクのある表現を回避しやすくなる。これも筆者が試した中で出力の質が変わりやすかった箇所だ。

## カスタム方法：3軸で差し替えるだけ

上記のプロンプトは「職種・評価段階・評価項目」を書き換えることで他の場面にも転用できる。

- **職種を変える**：「営業」→「企画」「人事」「現場リーダー」など
- **評価段階を変える**：「改善が必要」→「標準的」「良い」
- **評価項目を変える**：「正確性」→「コミュニケーション力」「主体性」「リーダーシップ」など

評価コメントの型（事実→評価→期待）と3点チェック（事実と一致しているか・断定表現がないか・本人が納得できるか）はどの職種でも共通して使える考え方なので、上記サンプルの構造を参考にしながら自分の職場に合わせて調整してみてほしい。
