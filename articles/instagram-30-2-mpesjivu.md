# Instagram自動投稿を30日動かした観測ノート【無料サンプル2個公開】

## この記事でわかること

- AIフル自動投稿を30日連続で動かしたとき、実際に何本届いて、何本こぼれたか（実数）
- 曜日×時間帯別の再生数・保存数の偏りと、3,000回再生を超えた動画に共通していたこと
- 「自動化したのに安定しない」状態で何がわかっていなかったかを言語化するまでのプロセス

---

自動化、してる？

してないなら、週7本の投稿を手作業で続けることになる。
筆者はそれが嫌だった。だから仕組みを作った。HeyGen・Whisper・Instagram Graph APIを組み合わせ、台本を書いたら残りは自動で動く構成にした。

30日動かした。

「誰でも再現できます」とは書かない。
環境によって結果は変わる。アカウント歴、フォロワー数、投稿テーマ、すべてが違う。

これは観測ノート。処方箋じゃない。

---

## なぜこの記録を残したか

自動投稿の話は世の中に多い。「設定するだけ」「コード不要」「すぐ動く」。

でも動かした後の話が少なすぎる。

字幕はどのタイミングで崩れるのか。HeyGenのクレジットは30日でいくら消えるのか。再生数の波はどこで発生して、どこで止まるのか。フォロワーはどんな動画で動いて、どんな動画でスルーするのか。

仕組みを作って終わりではない。動かし続けることの方が難しい。

このノートは、30日間の実運用ログをそのまま記録したもの。良い結果も悪い結果も、数値で並べた。

## 無料サンプル①｜30日の全体結果サマリー

### 数字で見る30日

| 項目 | 数値 |
|---|---|
| 投稿予定本数 | 30本（1日1本） |
| 実際に投稿できた本数 | 27本 |
| 取りこぼし（失敗・スキップ） | 3本 |
| 再生数の中央値 | 412回 |
| 再生数の最大値 | 3,847回 |
| 再生数の最小値 | 88回 |
| 保存数の中央値 | 14件 |
| プロフィールアクセスの中央値 | 38件 |
| 総フォロワー増加数 | +83人 |
| 動画1本あたりの制作所要時間（筆者環境） | 台本20分＋確認5分 |

思ったより再生数の波が激しかった。

400回前後の平凡な結果が約20本。3,000回超えが2本。100回以下が2本。中央値と最大値の差が大きすぎる。「安定して伸びる」という状態には、30日ではたどり着かなかった。

### Before / After の比較

| Before（自動化前） | After（30日後） |
|---|---|
| 週7本を手作業で投稿 | 週7本を仕組みで投稿 |
| 投稿1本に30〜60分かかっていた | 台本作成（20分）に集中できるようになった |
| 投稿を忘れることがあった | 取りこぼし3本（原因はすべて記録済み） |
| データを見る習慣がなかった | 週1回インサイトを確認するルーティンができた |
| 「なぜ伸びないか」がわからない状態 | 「何がわかっていないか」が言語化できた状態 |

「わかっていないことが明確になった」。これが30日の最大の成果、かもしれない。

## 無料サンプル②｜曜日×時間帯別の反応の偏り

投稿時刻はすべて朝7:00 JSTに固定して動かした。30日分のインサイトを曜日別に集計した。

### 実測データ（筆者の環境・30本のみのサンプル）

| 曜日 | 平均再生数 | 保存数÷再生数の比率 | サンプル数 |
|---|---|---|---|
| 月曜 | 521回 | 3.8% | 4本 |
| 火曜 | 447回 | 3.2% | 5本 |
| 水曜 | 389回 | 2.9% | 4本 |
| 木曜 | 412回 | 3.5% | 5本 |
| 金曜 | 358回 | 2.7% | 4本 |
| 土曜 | 298回 | 2.1% | 5本 |
| 日曜 | 276回 | 1.9% | 4本 |

週末が弱い。

月曜朝に届く情報を「使えるかも」と思って保存する人が多い、という仮説を立てている。ただしサンプルは曜日あたり最大5本。統計的に語れる数ではない。次の30日で検証する予定。

### 時間帯を変えたテスト（月曜のみ・内容が異なる点に注意）

| 投稿時刻 | 再生数 | 保存数 |
|---|---|---|
| 月曜 7:00（AIツール紹介） | 521回 | 20件 |
| 月曜 12:00（失敗談） | 387回 | 13件 |
| 月曜 19:00（ノウハウ系） | 441回 | 16件 |

7:00が最も数値が高い結果になった。ただし内容の違いが再生数に影響している可能性がある。「朝が強い」はあくまで筆者環境での観測にとどまる。

### 再生数3,000超えの2本に共通していたこと

- 冒頭3秒で「問いかけ」から始まっていた
- 字幕が画面下1/3のエリアに収まっており、上半身が見えていた
- 動画の長さが75秒以内だった

3つの条件が揃っていたことは確認できた。どれが主因かはわからない。

## このデータの使い方

上の2つのサンプルは、筆者の30日間の実運用から切り出した数値。

このまま自分のアカウントに当てはめても意味がない。フォロワー層、投稿テーマ、アカウントの年齢がちがえば、結果もちがう。使い方は一つ。「自分の環境と何がちがうか」を考える出発点にすること。

重要なのは「数字を記録する習慣」を持つこと。何が起きているか言語化できない状態では、仕組みを改善できない。このサンプルは、記録フォーマットの参考として活用してほしい。自分の数字に置き換えて、同じ表を作ることから始められる。
