## 「うまく書けない」は問題じゃない

コンテンツ制作で時間を食われる理由。
多くの人が「文章力が足りない」と思っている。違う。

本当の問題は、**AIに何をどう頼めばいいか、わかっていないこと**。

「メルマガを書いて」とだけ投げても、AIは動けない。
読者は誰か。悩みは何か。CTAはどこへ向けるか。
この情報がなければ、出てくるのは当たり障りのない文章だけ。

プロンプトに「型」を持つと、状況が変わる。
筆者（蒼井詠）の場合、メルマガ1本あたりの作業時間が体感で大幅に短縮された。ただし、これは筆者の環境・運用歴・スキルセットが前提。誰でも同じ結果になるわけではない。

この記事では、実際に運用しているプロンプト集から2本を無料公開する。
コピペして`【　】`だけ書き換えれば、今日から使える。

## 無料サンプル① メルマガ件名を10本量産するプロンプト

```
あなたはメールマーケティングの専門家です。
以下の条件でメルマガ件名を10本作ってください。

【テーマ】〇〇（例：ChatGPTで議事録を自動化する方法）
【読者の悩み】〇〇（例：会議のたびに30分かけてメモを清書している）
【読者の属性】〇〇（例：中小企業の営業職・30代・PC操作は普通レベル）

条件：
- 開封率を意識し、数字・問いかけ・意外性の3パターンを各3本以上含める
- 件名は20文字以内
- 「無料」「絶対」「必ず」などの誇張表現は使わない
- 最後に「使い分けの判断基準」を一言添える

出力形式：番号付きリスト
```

### 入力例

- **テーマ**：ChatGPTで議事録を自動化する方法
- **読者の悩み**：会議のたびに30分かけてメモを清書している
- **読者の属性**：営業職・30代・PC操作は普通レベル

### 出力例（抜粋）

> 1. 議事録、まだ手書きですか？
> 2. 30分の作業が5分になった話
> 3. 会議後に何もしない人がいる理由
> 4. ChatGPTに会議を任せてみた
> 5. 「書く時間」に価値はあるのか

### ChatGPT vs Claude の差

| | ChatGPT | Claude |
|---|---|---|
| 件名の傾向 | バリエーションが豊富。エモーショナルな表現が多め | 論理的・問いかけが鋭い。B2B向きの文体に強い |
| 向いている用途 | BtoCの感情訴求コンテンツ | BtoBや教育系・丁寧な説明が必要な内容 |

どちらが優れているかではなく、読者層に合わせて使い分けるのが実用的な判断。

## 無料サンプル② LPヒーローセクションを生成するプロンプト

LP（販売ページ）は、セクションをまとめて一括生成しようとすると品質が落ちる。
パーツ単位で分けて生成し、あとで組み合わせる方法が安定する。

まず最上部のヒーローセクションから始める。

```
LP（販売ページ）の最上部に置くヒーローセクションを書いてください。

【商品・サービス名】〇〇
【誰向けか（ターゲット）】〇〇（例：副業で情報発信を始めた会社員）
【解決する悩み】〇〇（例：毎週のブログ・メルマガ・SNSのネタ出しに時間がかかる）
【提供する価値】〇〇（例：AIプロンプト集を使えばネタ出しと下書きが一セットで完成する）

出力してほしいもの：
1. キャッチコピー（25字以内・インパクトのある1行）
2. サブキャッチ（60字以内・キャッチを補足する1文）
3. ボディコピー（3〜5行・ターゲットの悩みと解決策を結ぶ文章）
4. CTAボタン文言（例：「いますぐ手に入れる」「無料サンプルを見る」）

誇張・断定的な効果訴求（〇〇%改善・必ず売れる など）は使わない。
体験・比較・変化の描写にとどめること。
```

### 比較：プロンプトなしで書いた場合 vs プロンプト使用

| | プロンプトなし | プロンプト使用 |
|---|---|---|
| キャッチ | 「AIで作業効率化！」 | 「ネタ切れ、今日で終わりにする」 |
| 印象 | 説明的・ありきたり | 読者の課題を直撃している |

プロンプトを使ったからといって毎回良い出力になるわけではない。
ただ、「何を書くべきか」の軸が定まっている分、修正作業がやりやすくなる、というのが筆者の体感。

## カスタマイズの手順

このプロンプトはジャンルを問わず使える設計にしている。

1. **`【　】`の中だけを自分のサービス情報に書き換える**
   テンプレの構造は変えなくていい。

2. **出力が気に入らなければ「条件」を1行追加する**
   例：「箇条書きは使わない」「語尾を体言止めにする」など。

3. **ChatGPT と Claude 両方で同じプロンプトを試す**
   出力のトーンが異なる。ターゲットに合う方を選ぶ。

修正・調整前提で使うのが現実的。
「プロンプトを送ったら完成品が出てくる」という期待値では、どのツールも使いにくい。
