## 文章が苦手な発信者のリアルな課題

SNSを継続するうえで、地味に時間を奪われるのが「キャプション文」の作成だ。

動画を撮る。編集する。ここまでは頑張れる。でもいざキャプションを書こうとすると——

> 「なんか言葉がふわっとしてる気がする」
> 「毎回テイストがバラバラで統一感がない」
> 「書いても書いても、何か違う」

AIを使えば解決できると思い、ChatGPTに「Instagram投稿文書いて」と打ち込んでも、返ってくるのはどこかのマーケター風の模範文。自分のキャラクターじゃない。

原因は明確。プロンプトに「**文体の定義**」が入っていないから。

---

## 文体定義があるプロンプトとないプロンプトの差

AIは「何を書くか」より「**どう書くか**」の指示に反応しやすい。

**曖昧な指示：**
```
Instagramのキャプションを書いてください。テーマはAI自動化です。
```

**文体定義ありの指示：**
```
断定口調・体言止め・短文（1文30字以内）でInstagramキャプションを書いてください。
感情表現・お願い口調は禁止。テーマはAI自動化。
```

同じAI、同じテーマ。出力は別物になる。

「蒼井詠スタイル」と呼ぶのは、後者の設計思想を言語化したもの。断定・クール・体言止め——この3原則をプロンプトに組み込むことで、出力が安定する。

## 無料サンプル3本

### サンプル① 蒼井詠文体マスタープロンプト

全SNS共通の文体ルールをAIに設定する「初期化プロンプト」。会話の冒頭に貼り付けるだけで、その後の出力が変わる。ChatGPT・Claude・Geminiどれでも動作を確認している。

```
あなたは「蒼井詠」というAIキャラクターの文体でSNS投稿文を書くアシスタントです。
以下の文体ルールを厳守してください。

【文体ルール】
- 断定口調：「〜です」「〜します」より「〜だ」「〜する」を優先
- 体言止め：文末を名詞や動名詞で締める（例：「これが正解。」「非効率は罪。」）
- 感情を表に出さない：共感・謝罪・配慮表現を最小化
- 短文主義：1文は30字以内を目安にする
- 具体性：曖昧な表現より数値・固有名詞・動詞を優先

【禁止表現】
- 「〜かもしれません」「〜でしょうか」などの曖昧語
- 「ぜひ」「よかったら」などのお願い口調
- 「素晴らしい」などの感情的な誉め言葉

これらのルールを確認したら「了解です」とだけ返答してください。
```

### サンプル② 断定口調・体言止め変換プロンプト

過去に書いた記事、他SNSの文章、自分のメモ——既存テキストを蒼井詠スタイルに変換するリライト専用プロンプト。`{}`の中に変換したい文章を貼り付けるだけ。

```
以下の文章を「蒼井詠スタイル」にリライトしてください。

【蒼井詠スタイルのルール】
- 断定口調（「〜だ」「〜する」）
- 体言止めを積極的に使う
- 1文30字以内を目安に短く切る
- 曖昧語・お願い口調・感情表現を削除
- ハッシュタグは元の文にあれば保持

【リライト対象】
{ここに変換したい文章を貼り付ける}

出力形式：リライト後のテキストのみ（解説不要）
```

出力が返ってきたら、そのままコピーして投稿できる形式で出てくる。「解説不要」の一言が余計な出力を抑制するポイント。

### サンプル③ Instagram Reelキャプション（ノウハウ解説系）

AIツールや自動化ノウハウを紹介するReel動画用のキャプション生成プロンプト。テーマとノウハウの概要を入力すると、フック文・本文・CTAの構成で出力される。

```
あなたはSNSキャプションの専門家です。
蒼井詠スタイル（断定・クール・体言止め）でInstagram Reelのキャプションを生成してください。

【入力情報】
- 動画テーマ：{テーマを入力}
- 動画で紹介するノウハウ：{ノウハウの概要を入力}
- ターゲット：AI・副業に興味がある20〜40代

【生成ルール】
1. 冒頭1行：思わず止まるフック文（疑問形または断定）
2. 本文3〜5行：ノウハウの核心を箇条書きまたは短文で展開
3. CTA最終行：保存・フォローを促す一言（蒼井詠口調で）

【文体ルール】
- 体言止め・断定口調で統一
- 1行30字以内
- 「ぜひ試してみてください」など柔らかいお願い表現禁止

ハッシュタグは生成しない（別途追加する）
```

テーマに「Notionの自動化」、ノウハウに「データベース連携でタスク管理を半自動にする手順」を入れると、そのままReel説明欄に貼れる文章が出力される。

## カスタム方法

3本のサンプルには共通の構造がある。

1. **文体ルールのセクション**（断定・体言止め・禁止表現の定義）
2. **入力変数のセクション**（`{}`で囲った差し替え部分）
3. **出力形式のセクション**（「解説不要」「テキストのみ」などの制約）

この3層構造を守れば、キャラクターを自分に変えても同じ設計で動く。「蒼井詠スタイル」の名称と口調サンプルを自分のキャラクター設定に書き換えるだけで、プロンプトの骨格はそのまま流用できる。

AIへの指示が曖昧なほど出力がブレる。変数と禁止事項を明示するほど、出力の再現性が上がる——これがプロンプト設計の基本原則。自分のキャラクターを育てている発信者にとっては、文体の言語化そのものが資産になる。
